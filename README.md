# zh-index-pages

本仓库用于归档和发布多个独立的 HTML 页面。这些页面内容彼此独立，不依赖于特定域名或后端服务，可直接在浏览器中打开运行。

## 项目简介

`zh-index-pages` 是一个静态 HTML 页面集合仓库。主要用途包括：

- 存放与维护各类独立 HTML 页面
- 提供页面归档与版本管理
- 方便通过 GitHub Pages 或其他静态托管服务进行发布

仓库本身不针对任何特定网站或服务，仅为页面存档与分发提供基础设施。

## 目录说明

```
├── index.html          # 仓库首页 / 页面导航（如有）
├── pages/              # 存放各独立 HTML 页面的目录
│   ├── example.html    # 示例页面（可删除或替换）
│   └── ...
├── assets/             # 公共资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：每个文件为一个完整的 HTML 页面，建议使用有意义的文件名。
- `assets/`：存放页面间共享的样式、脚本或图片资源。若页面完全自包含，可忽略此目录。

## 页面归档说明

- 每个 HTML 页面均为独立归档单元，可包含完整的内联样式与脚本。
- 页面之间不建议相互依赖，以保证单页可独立运行。
- 归档内容应遵守相关法律法规，不包含恶意代码、侵犯性内容或违反 GitHub 服务条款的内容。

## 维护说明

- 提交新页面时，请在 `pages/` 目录下添加对应的 HTML 文件。
- 更新已有页面时，建议在提交信息中简要说明变更内容。
- 删除页面时，请确认无其他页面引用其资源。
- 若启用 GitHub Pages，仓库根目录或 `docs/` 目录可配置为发布源，`index.html` 将作为默认入口。
